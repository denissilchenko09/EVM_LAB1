# EVM_LAB1
Данная программа генерирует учебные варианты для демонстрации разницы в точности представления вещественных чисел в форматах float (32 бита), double (64 бита) и __float128 (128 бит, quad precision) стандарта IEEE 754.

Для каждого случайно сгенерированного числа (с большим количеством знаков после запятой) программа вычисляет его представление в трёх типах, выводит битовое представление (знак, порядок, мантисса) и абсолютную погрешность по сравнению с исходной строкой.
Возможности

    Генерация заданного количества вариантов, в каждом варианте — заданное количество чисел.

    Для каждого числа сохраняется:

        Исходное число (в виде строки с 100 знаками после запятой).

        Битовое представление IEEE 754 для float, double и __float128.

        Абсолютная погрешность (разница между исходной строкой и округлённым значением, представленным в виде строки).

    Создаются файлы:

        variant_1.txt, variant_2.txt, … — только исходные числа (для выдачи ученикам).

        answers.txt — подробная таблица с ответами для преподавателя.

Требования

    Компилятор GCC (поддержка __float128 и библиотеки quadmath).

    Библиотека quadmath (обычно устанавливается вместе с GCC или отдельно, например, libquadmath0).

Компиляция
bash

gcc -o float_precision_demo source.c -lquadmath

Замените source.c на имя вашего исходного файла.
Использование
bash

./float_precision_demo <количество вариантов> <чисел в варианте>

Пример:
bash

./float_precision_demo 5 3

Будут созданы:

    5 файлов variant_1.txt … variant_5.txt, в каждом по 3 числа.

    Файл answers.txt с подробными ответами.

Формат выходных файлов
Файлы для учеников (variant_N.txt)

Каждая строка содержит одно число в виде 0.123456… (ровно 100 знаков после запятой).
Файл для учителя (answers.txt)

Для каждого варианта и числа выводится таблица:
text

=== ВАРИАНТ 1 ===

+-------------------------------------------------------------------------------+
| Число 1: 0.12345678901234567890123456789012345678901234567890123456789012... |
+-----------------+-----------------------------------------------------------------------------+----------------------+
| Тип             | Битовое представление (IEEE 754)                                             | Абсолютная погрешность |
+-----------------+-----------------------------------------------------------------------------+----------------------+
| 32-bit (float)  | 0 01111100 10011100001001001101111                                           | 0000000000000000...   |
| 64-bit (double) | 0 01111111100 1001110000100100110111100101101111100111011011010101           | 0000000000000000...   |
| 128-bit (quad)  | 0 011111111111100 1001110000100100110111100101101111100111011011010101...    | 0000000000000000...   |
+-----------------+-----------------------------------------------------------------------------+----------------------+

Погрешность представлена в виде строки цифр (поразрядная разность с исходным числом). Нули означают точное совпадение разряда.
Особенности реализации

    Используется библиотека quadmath.h для работы с __float128.

    Генерация случайных чисел: каждый разряд после запятой выбирается случайно от 0 до 9.

    Битовое представление извлекается путём копирования байтов и печати битов с разделением на знак, порядок и мантиссу согласно стандарту IEEE 754.

    Погрешность вычисляется как абсолютная разность двух строк, изображающих десятичные дроби (поразрядно, с учётом заёма).

Примечания

    Программа предназначена для учебных целей, чтобы наглядно показать ограниченность машинного представления вещественных чисел.

    Для корректной работы требуется компилятор с поддержкой __float128 (GCC 4.6+).

    Если ваша система не поддерживает __float128, можно заменить его на long double (обычно 80 бит на x86) и удалить зависимость от quadmath.h. В этом случае точность будет ниже.

    При компиляции на некоторых системах может потребоваться явно указать путь к библиотеке quadmath (флаги -L).
